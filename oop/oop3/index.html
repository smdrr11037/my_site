    函数调用有代价（栈等）

    Inline Function

    将 function 在调用处展开，不用调用和返回，没有函数调用的开销了（坏处是代码变大）

    用 inline 关键字：

    // 需要调用的版本
int f(int i){
    return i*2;
}
main(){
    int a=4;
    int b=f(a);
}

// 原地转换的版本
inline int f(int i){
    return i*2;
}
main(){
    int a=4;
    int b=f(a);
}
    • 类似于宏，但是宏是简单的字符串替换，inline 具备语法检查的功能
    • inline 函数的定义要放在头文件里，否则编译器无法对齐展开
    • 可能不会在 .obj 文件中产生任何代码
    • inline 会有类型转换和返回值

    宏和 inline
    * 函数写在 class 声明里，认为是 inline * 如果想这么干，可以在类外面这么写，便于看清类的接口

    继承

    继承的目的：将子类的共性提取为父类，使得父类的代码可以重用

    接口重用，使用的代码易合并

    子类具有父类的所有接口（创建时会复制一份）

    继承的代码示例
    class Item
{

}

class CD: public Item// 继承标识
{

}

    • public 和 protect

      • protect：子类可见，客户不可见

    • 父类需要先构造（不加参则调用缺省构造函数）

    • 析构先调用子类

    • 输入输出流可以作为方法的参数

    输入输出流作为参数
    inline void Employee::print(std::ostream& out) const

    子类的同名函数会覆盖父类的函数（overwrite）

    Vector Class

    vector 类的使用例子
    #include <iostream>
#include <vector>

int main() {
    std::vector<int> numbers; // 需要 std 命名空间
    // 不需要担心 size

    // Add elements to the vector
    numbers.push_back(10);
    numbers.push_back(20);
    numbers.push_back(30);

    // 可以用标号访问
    std::cout << "First element: " << numbers[0] << std::endl;

    // 迭代器访问（与操作符重载有关）
    vector<int>::iterator p;
    for(p=numbers.begin();p<=numbers.end();p++){
        // 访问每个元素
    }
    return 0;
}
    • Constructors: vector<Elem> c; vector<Elem> c1(c2);
    • Add/Remove/Find: V.find(...) - 返回迭代器类型
